🚀 Stack Complète pour Héberger des Applications Python

Nginx, Uvicorn/Gunicorn, systemd, Certbot, ufw, fail2ban

🎯 Objectif Héberger plusieurs sites et applications Python sur un même serveur de manière rapide, sécurisée et stable

🌐 Flux d'une Requête HTTP/HTTPS

🌍
Internet
L'utilisateur envoie une requête HTTP/HTTPS depuis son navigateur
Protocoles :
HTTP (port 80)
HTTPS (port 443)
↓
🔥
ufw (Pare-feu)
Filtre les connexions entrantes et autorise uniquement les ports nécessaires
Ports ouverts :
22 (SSH), 80 (HTTP), 443 (HTTPS)
Tous les autres ports : fermés
🔀
Nginx (Reverse Proxy)
Serveur web frontal qui reçoit toutes les requêtes et les distribue
Rôles :
• Servir fichiers statiques (HTML, CSS, JS, images)
• Router vers les apps Python (FastAPI, Flask)
• Gérer HTTPS et certificats SSL
• Virtual hosts pour plusieurs sites
⚡
Uvicorn / Gunicorn
Serveur d'application ASGI/WSGI qui exécute le code Python
Choix :
• Uvicorn : Rapide, moderne (ASGI) pour FastAPI
• Gunicorn : Éprouvé, stable (WSGI) pour Flask
Communication via socket UNIX ou port local
↓
🐍
Application Python
Votre code (FastAPI, Flask, Django) qui génère les réponses
Frameworks supportés :
FastAPI, Flask, Django, Starlette
Retourne JSON, HTML, dashboards...

🛠️ Composants de Gestion et Sécurité

⚙️

systemd

Gestionnaire de services du système qui assure que vos applications tournent en permanence

Fonctions :
• Démarrage automatique au boot
• Redémarrage automatique si crash
• Gestion des logs via journalctl
• Contrôle via systemctl
🔒

Certbot + Let's Encrypt

Automatisation de la création et du renouvellement des certificats SSL/TLS pour HTTPS

Avantages :
• Certificats gratuits et valides
• Renouvellement automatique (90 jours)
• Configuration auto de Nginx
• Validation de domaine automatique
🛡️

fail2ban

Protection anti-bruteforce qui surveille les logs et bannit les IPs malveillantes

Protection :
• Détecte les tentatives SSH répétées
• Bannit automatiquement via iptables/ufw
• Surveille Nginx, SSH, et autres services
• Configuration automatique

🔀 Configuration Nginx (Reverse Proxy)

Fichiers statiques

Nginx sert directement les fichiers HTML, CSS, JS, images sans passer par Python

location /static/ {
  alias /srv/apps/dashboard1/static/;
  expires 30d;
}

Proxy vers Python

Les requêtes dynamiques sont transmises à l'app Python via socket UNIX

location / {
  proxy_pass http://unix:/srv/apps/dashboard1/run/uvicorn.sock;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
}

HTTPS automatique

Certbot configure automatiquement SSL et les redirections HTTP → HTTPS

listen 443 ssl http2;
ssl_certificate /etc/letsencrypt/live/monsite.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/monsite.com/privkey.pem;

⚙️ Configuration systemd (Gestion des Services)

Exemple de service systemd pour une application FastAPI

# /etc/systemd/system/dashboard1.service [Unit] Description=Dashboard Application (FastAPI) After=network.target [Service] Type=notify User=www-data Group=www-data WorkingDirectory=/srv/apps/dashboard1 Environment="PATH=/srv/apps/dashboard1/venv/bin" ExecStart=/srv/apps/dashboard1/venv/bin/uvicorn app:app \ --uds /srv/apps/dashboard1/run/uvicorn.sock \ --workers 4 Restart=always RestartSec=5 [Install] WantedBy=multi-user.target

Commandes essentielles

# Activer le service au démarrage
sudo systemctl enable dashboard1

# Démarrer le service
sudo systemctl start dashboard1

# Voir le statut
sudo systemctl status dashboard1

# Redémarrer après modification
sudo systemctl restart dashboard1

# Voir les logs
sudo journalctl -u dashboard1 -f

Avantages de systemd

  • Démarrage automatique au boot du serveur
  • Redémarrage automatique en cas de crash
  • Gestion centralisée des logs
  • Contrôle des ressources (CPU, mémoire)
  • Dépendances entre services

🔒 Sécurité : ufw + fail2ban + HTTPS

ufw (Uncomplicated Firewall)

Pare-feu simple qui contrôle les connexions entrantes

# Autoriser SSH, HTTP, HTTPS
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Activer le pare-feu
sudo ufw enable

# Voir le statut
sudo ufw status

fail2ban

Détection et blocage automatique des tentatives d'intrusion

# Installation
sudo apt install fail2ban

# Configuration auto pour SSH et Nginx
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# Voir les IPs bannies
sudo fail2ban-client status sshd

Certbot (HTTPS)

Installation automatique des certificats SSL

# Installation
sudo apt install certbot python3-certbot-nginx

# Générer le certificat
sudo certbot --nginx -d monsite.com -d www.monsite.com

# Renouvellement auto (cron)
sudo certbot renew --dry-run

⚡ Uvicorn vs Gunicorn : Quel serveur choisir ?

✅ Uvicorn (recommandé pour FastAPI)

  • ASGI : Support natif des WebSockets et async/await
  • Rapide : Hautes performances pour les apps modernes
  • Léger : Consommation mémoire optimisée
  • Moderne : Conçu pour Python 3.7+
uvicorn app:app --uds /run/app.sock --workers 4

✅ Gunicorn (classique pour Flask)

  • WSGI : Standard éprouvé depuis des années
  • Stable : Production-ready avec gestion des workers
  • Compatible : Flask, Django, Pyramid
  • Options : Nombreuses options de configuration
gunicorn app:app --bind unix:/run/app.sock --workers 4

💡 Conseil : FastAPI/Starlette → Uvicorn | Flask/Django → Gunicorn

✅ Checklist d'Installation Complète (Ubuntu 22.04 / Debian 12)

  • Mise à jour du système : sudo apt update && sudo apt upgrade
  • Installation Nginx : sudo apt install nginx
  • Installation Python et venv : sudo apt install python3 python3-pip python3-venv
  • Création environnement virtuel : python3 -m venv /srv/apps/mon-app/venv
  • Installation FastAPI/Flask : pip install fastapi uvicorn (ou flask gunicorn)
  • Configuration service systemd : Créer /etc/systemd/system/mon-app.service
  • Activation du service : sudo systemctl enable mon-app && sudo systemctl start mon-app
  • Configuration Nginx : Créer /etc/nginx/sites-available/mon-app.conf
  • Activation du site : sudo ln -s /etc/nginx/sites-available/mon-app.conf /etc/nginx/sites-enabled/
  • Test Nginx : sudo nginx -t && sudo systemctl reload nginx
  • Installation Certbot : sudo apt install certbot python3-certbot-nginx
  • Génération certificat SSL : sudo certbot --nginx -d monsite.com
  • Configuration ufw : sudo ufw allow 22/80/443 && sudo ufw enable
  • Installation fail2ban : sudo apt install fail2ban && sudo systemctl enable fail2ban
  • Test final : Accéder à https://monsite.com et vérifier que tout fonctionne

📚 Lexique des Termes Techniques

Reverse Proxy

Serveur intermédiaire qui reçoit les requêtes des clients et les transmet aux serveurs backends. Nginx joue ce rôle en distribuant les requêtes vers les apps Python.

ASGI / WSGI

ASGI (Asynchronous Server Gateway Interface) : Standard moderne supportant async/await et WebSockets. WSGI (Web Server Gateway Interface) : Standard plus ancien, synchrone uniquement.

Socket UNIX

Fichier de communication local plus rapide qu'un port TCP. Nginx et Uvicorn communiquent via /run/app.sock au lieu de localhost:8000.

Virtual Host

Configuration Nginx permettant d'héberger plusieurs sites/domaines sur le même serveur. Chaque domaine a sa propre config dans /etc/nginx/sites-available/.

systemd

Gestionnaire de services et d'initialisation présent sur la plupart des distributions Linux modernes (Ubuntu, Debian, Fedora, etc.). Remplace les anciens scripts init.d.

Let's Encrypt

Autorité de certification gratuite qui délivre des certificats SSL/TLS valides et reconnus par tous les navigateurs. Renouvellement automatique tous les 90 jours via Certbot.

Certbot

Outil en ligne de commande qui automatise la génération, l'installation et le renouvellement des certificats Let's Encrypt. S'intègre directement avec Nginx.

ufw

Uncomplicated Firewall : Interface simplifiée pour iptables qui permet de gérer facilement les règles de pare-feu sur Ubuntu/Debian.

fail2ban

Service qui surveille les logs système (SSH, Nginx, etc.) et banni automatiquement les adresses IP qui présentent des comportements malveillants (tentatives répétées de connexion).

Workers

Processus parallèles qui exécutent votre application. Plus de workers = plus de requêtes simultanées. Règle générale : 2-4 workers par cœur CPU.

journalctl

Commande pour consulter les logs systemd. Permet de voir en temps réel ce qui se passe dans vos services (erreurs, démarrages, crashs).

HTTP/2

Version moderne du protocole HTTP qui améliore les performances (multiplexage, compression des headers). Activé automatiquement par Certbot dans Nginx.

🎯 En résumé : Stack Production en 6 Couches

1. 🌍 Internet envoie requête → 2. 🔥 ufw filtre les ports → 3. 🔒 Nginx gère HTTPS et route → 4. ⚡ Uvicorn/Gunicorn exécute Python → 5. 🐍 Ton code génère la réponse → 6. ⚙️ systemd supervise tout

🔑 Le principe clé : Chaque couche a un rôle précis et communique avec les autres de manière optimisée !

🛡️ Sécurité : fail2ban + ufw + HTTPS (Certbot) = Serveur protégé et professionnel
