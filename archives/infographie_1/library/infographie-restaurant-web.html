🍽️ L'Application Web expliquée par un Restaurant

Comprendre l'architecture web en 5 minutes

👤
Le Client
= La personne affamée
Navigateur Web
Chrome, Firefox, Safari

Il envoie des requêtes HTTP/HTTPS (= les commandes du client)
→
📡
Le Routeur
= Le livreur qui trouve l'adresse
Internet / DNS
Traduit "monsite.com" en adresse IP
Achemine la requête au bon serveur
↓
🛡️
Nginx
= Le videur / Maître d'hôtel
Serveur Web / Reverse Proxy
• Sert les fichiers statiques (HTML, CSS, JS, images)
• Gère le HTTPS (certificat SSL)
• Filtre les requêtes malveillantes
• Redirige vers Flask si besoin
⚡ Pour les demandes simples (menu = page HTML), Nginx répond direct
⚡ Pour les demandes complexes (plat cuisiné = données dynamiques), il appelle Flask
↓
👨‍🍳
Flask (Backend)
= Le chef cuisinier
Framework Python / app.py
• Traite la logique métier
• Gère les routes/API
• Génère le contenu dynamique
• Communique avec la base de données
• Écoute sur 127.0.0.1:5000 (caché)
↔
🗄️
Base de Données
= Le frigo / La réserve
PostgreSQL / MySQL / SQLite
Stocke les données :
• Utilisateurs
• Articles
• Commandes
• etc.
↓
🖥️
Le Navigateur reçoit la réponse
= Le client reçoit son plat
HTML + CSS + JavaScript
• HTML = structure (l'assiette)
• CSS = style (la présentation)
• JavaScript = interactions (animations, formulaires)

📚 Lexique des termes techniques

HTTP / HTTPS

HyperText Transfer Protocol

Le langage utilisé par le navigateur et le serveur pour communiquer. HTTPS = version sécurisée (cadenas 🔒)

GET /menu → "Donne-moi la page menu"

Requête HTTP

La demande envoyée par le navigateur au serveur

GET = lire une page
POST = envoyer des données (formulaire)
PUT = modifier
DELETE = supprimer

HTML

HyperText Markup Language

Le squelette de la page : titres, paragraphes, images, liens

<h1>Bienvenue</h1>

CSS

Cascading Style Sheets

La décoration : couleurs, polices, mise en page

h1 { color: blue; }

JavaScript

Le langage qui rend la page interactive : animations, validation de formulaires, mises à jour sans recharger

button.onclick = () => alert('Coucou!');

API

Application Programming Interface

Un "menu technique" : points d'accès pour récupérer/envoyer des données en JSON

GET /api/users → liste des utilisateurs

Backend / Frontend

Backend = Cuisine (Flask, app.py, base de données)
Frontend = Salle (HTML, CSS, JS que voit le client)

Reverse Proxy

Nginx agit comme intermédiaire : il reçoit les requêtes et les redirige vers Flask en interne

Avantages : sécurité, cache, load balancing

app.py

Le fichier Python principal de Flask qui contient toute la logique de l'application

@app.route('/hello')
def hello():
  return 'Bonjour!'

Route

Un chemin d'URL qui déclenche une fonction spécifique dans Flask

/accueil → page d'accueil
/profil → page profil
/api/data → données JSON

Fichiers statiques

Fichiers qui ne changent pas : images, CSS, JavaScript. Nginx les sert directement sans passer par Flask = ultra rapide

127.0.0.1

localhost = l'adresse de la machine elle-même. Flask écoute sur cette adresse = invisible depuis Internet (protection !)

🎯 En résumé

Client (navigateur) → envoie requête HTTP → Routeur (Internet) → trouve le serveur → Nginx (videur) → sert fichiers statiques OU redirige vers → Flask (chef) → cuisine la réponse avec la Base de données (frigo) → retourne HTML/CSS/JS → Client reçoit la page 🎉
