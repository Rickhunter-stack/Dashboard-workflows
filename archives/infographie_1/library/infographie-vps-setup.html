🖥️ Mise en route d'un VPS non managé

Du serveur vierge à la production en 10 étapes

0
🏢
Le Fournisseur VPS
= Le constructeur qui livre l'appartement
OVH, DigitalOcean, Hetzner, etc.

Ce qu'ils livrent :
• Une partition du serveur physique
• Un OS Linux installé (Ubuntu, Debian, CentOS...)
• Les accès root (login + mot de passe)
• Une adresse IP publique
IP: 203.0.113.42
User: root
Password: Xk9$mP2#vL
↓
1
🔐
Première Connexion SSH
= Récupérer les clés de l'appartement
Connexion sécurisée au serveur

Depuis votre ordinateur :
ssh root@203.0.113.42
⚠️ Vous êtes maintenant root = administrateur tout-puissant
Tout ce que vous faites impacte le serveur directement
↓
2
📦
Mise à jour du système
= Vérifier que tout fonctionne et est à jour
Mettre à jour tous les paquets

apt update && apt upgrade -y
• apt update : récupère la liste des mises à jour
• apt upgrade : installe les mises à jour
• -y : accepte automatiquement
↓
3
👤
Créer un utilisateur non-root
= Ne pas utiliser le pass master tous les jours
Principe : ne JAMAIS travailler en root

adduser admin
usermod -aG sudo admin
• Crée l'utilisateur "admin"
• Lui donne les droits sudo (élévation temporaire)
• Vous travaillerez toujours avec cet utilisateur
↓
4
🔑
Configurer l'authentification par clé SSH
= Remplacer la clé physique par un badge sécurisé
Plus sécurisé qu'un mot de passe

Sur votre PC :
ssh-keygen -t ed25519
ssh-copy-id admin@203.0.113.42
• Génère une paire clé publique / privée
• Copie la clé publique sur le serveur
• Vous pourrez vous connecter sans mot de passe
↓
5
🛡️
Sécuriser SSH
= Verrouiller la porte et changer la serrure
Éditer /etc/ssh/sshd_config

PermitRootLogin no
PasswordAuthentication no
Port 2222
• Interdire connexion root
• Désactiver mot de passe (seulement clé SSH)
• Changer le port (optionnel mais recommandé)

systemctl restart sshd
↓
6
🔥
Configurer le Pare-feu
= Installer un système d'alarme
UFW (Uncomplicated Firewall)

ufw allow 2222/tcp # SSH
ufw allow 80/tcp # HTTP
ufw allow 443/tcp # HTTPS
ufw enable
• Bloque tout par défaut
• Autorise seulement les ports nécessaires
• Protège contre les attaques réseau
↓
7
🚫
Installer Fail2ban
= Caméra qui bannit les intrus
Protection contre le bruteforce

apt install fail2ban -y
systemctl enable fail2ban
• Surveille les logs de connexion
• Bannit les IP suspectes automatiquement
• Essais répétés = bannissement temporaire/permanent
↓
8
📊
Monitoring et Logs
= Installer des capteurs et caméras
Surveiller la santé du serveur

Outils essentiels :
• htop : voir CPU, RAM, processus
• journalctl : consulter les logs système
• netstat -tulpn : ports ouverts

Optionnel : Grafana, Prometheus, Netdata
↓
9
💾
Mettre en place les Sauvegardes
= Assurance en cas de catastrophe
Planifier des backups automatiques

• Snapshots du VPS (côté fournisseur)
• Backup des données importantes (rsync, borg)
• Backup de la base de données (mysqldump)
• Stockage externe : S3, autre serveur, NAS

Règle du 3-2-1 : 3 copies, 2 supports différents, 1 hors site
↓
10
🚀
Installer votre stack applicative
= Aménager l'appartement selon vos besoins
Maintenant vous pouvez installer :

• Serveur Web (Nginx, Apache)
• Base de données (PostgreSQL, MySQL, MongoDB)
• Runtime (Python, Node.js, PHP)
• Certificat SSL (Let's Encrypt / Certbot)
• Votre application (Flask, Django, Express...)

🎉 Votre serveur est prêt pour la production !

⚠️ Points de vigilance critiques

• TOUJOURS tester votre connexion SSH avant de fermer la session root
• VÉRIFIER que vous pouvez vous reconnecter avant de désactiver l'authentification par mot de passe
• NOTER vos identifiants et configurations dans un gestionnaire de mots de passe
• FAIRE un snapshot du VPS après chaque étape critique
• NE JAMAIS exposer des services non sécurisés sur Internet

📚 Lexique des termes techniques

VPS (Virtual Private Server)

Serveur privé virtuel : une partition d'un serveur physique qui fonctionne comme un serveur dédié indépendant.

Non managé = vous devez tout configurer vous-même

SSH (Secure Shell)

Protocole réseau crypté pour se connecter à distance et en toute sécurité à un serveur Linux.

ssh utilisateur@adresse-ip

Root

Le super-utilisateur sous Linux. Il a TOUS les droits sur la machine. Très puissant = très dangereux si mal utilisé.

⚠️ Une erreur en root peut détruire le système

sudo

Super User DO

Permet d'exécuter une commande avec les droits root temporairement.

sudo apt install nginx

Clé SSH

Paire de clés cryptographiques (publique/privée) pour s'authentifier sans mot de passe.

• Clé privée = garde secrète sur votre PC
• Clé publique = mise sur le serveur

Firewall / UFW

Pare-feu qui filtre le trafic réseau entrant et sortant. UFW = version simplifiée d'iptables.

Par défaut : tout bloquer, autoriser seulement le nécessaire

Fail2ban

Outil qui analyse les logs et bannit automatiquement les IP qui tentent des attaques (bruteforce, scan de ports, etc.)

Port

Numéro qui identifie un service réseau sur une machine.

22 (ou 2222) = SSH
80 = HTTP
443 = HTTPS
3306 = MySQL
5432 = PostgreSQL

apt (Advanced Package Tool)

Gestionnaire de paquets pour Debian/Ubuntu. Permet d'installer, mettre à jour et supprimer des logiciels.

apt update → MAJ liste paquets
apt upgrade → Installer les MAJ
apt install nginx → Installer Nginx

systemctl

Commande pour gérer les services (démarrer, arrêter, redémarrer, activer au démarrage)

systemctl start nginx
systemctl enable nginx
systemctl status nginx

Logs

Fichiers qui enregistrent tous les événements du système : connexions, erreurs, activités suspectes, etc.

/var/log/auth.log → Authentifications
/var/log/syslog → Système
journalctl → Logs systemd

Snapshot

Photo instantanée de l'état complet du VPS à un moment donné. Permet de restaurer en cas de problème.

💡 À faire AVANT toute modification critique

🎯 Checklist de mise en production

✅ Mise à jour système (apt update && upgrade)
✅ Utilisateur non-root créé avec sudo
✅ Authentification SSH par clé uniquement
✅ Connexion root SSH désactivée
✅ Firewall UFW configuré et activé
✅ Fail2ban installé et actif
✅ Monitoring en place (logs + métriques)
✅ Sauvegardes automatiques configurées
✅ Snapshot de sécurité créé
✅ Documentation des configurations sauvegardée
