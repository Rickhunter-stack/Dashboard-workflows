'': () => import('./pages/home/controller.js')
.then(m => m.render()),

.then(m => m.render())

Gestion d'État & Événements
= Le système nerveux de l'application
Mini store Pub/Sub suffit :

store.on('filters:change', callback)
store.emit('filters:change', data)

• Les composants s'abonnent aux changements
• Découplage fort = maintenance facile
• État global évènementiel
• Pas besoin de Redux ou MobX !

Sécurité : Pas de Secrets Client
= Ne jamais laisser les clés dans la voiture
RÈGLE ABSOLUE :

⛔ JAMAIS de secrets côté client
• Pas de clés API sensibles
• Pas de tokens privés
• Pas d'URL avec credentials

✅ Solution :
• Proxy serveur pour appels sensibles
• Côté client = données publiques uniquement
• Variables d'environnement côté serveur

Protection XSS & Injections DOM
= Filtrer l'eau avant de la boire
CRITIQUE : Protéger contre les injections

⛔ À ÉVITER :
• innerHTML avec données utilisateur
• eval(), new Function()
• innerHTML += (toujours dangereux)

✅ À UTILISER :
• textContent par défaut
• createElement pour le DOM
• DOMPurify si HTML nécessaire (vendored localement)
• Sanitization stricte de tout input utilisateur

Content-Security-Policy (CSP)
= Le videur à l'entrée du club
CSP stricte en production :

<meta http-equiv="Content-Security-Policy"
content="default-src 'self';
img-src 'self' data:;
style-src 'self';
script-src 'self';
connect-src 'self' https://api.example.com;
object-src 'none';
frame-ancestors 'none'">

• Bloque scripts non autorisés
• Protège contre XSS
• Idéalement en header HTTP (sinon meta)

Stockage Sécurisé
= Coffre-fort vs carnet sur la table
localStorage ≠ sécurisé !

⛔ Ne JAMAIS stocker :
• Tokens sensibles
• Mots de passe
• Données personnelles sensibles

✅ Bonnes pratiques :
• Cookies Secure + SameSite=Strict
• Cookies HttpOnly pour tokens (backend)
• localStorage OK pour préférences UI non sensibles
• Sessions côté serveur pour authentification

Couche de Sécurité Complète
= Sécurité multicouche
Checklist sécurité :

• HTTPS partout (obligatoire pour Service Worker)
• SRI : Subresource Integrity sur scripts tiers
• X-Frame-Options: DENY (anti-clickjacking)

Déploiement Simple
= Livrer le produit fini
Static hosting professionnel :

• GitHub Pages, Netlify, Vercel, S3+CloudFront
• Headers HTTP configurés (CSP, HSTS, etc.)
• Optionnel : mini build local (Rollup/Vite)
• Concaténation + minification → /dist

Avantages :
✅ Pas de runtime Node en prod
✅ Hébergement gratuit ou très bon marché
✅ CDN automatique
✅ HTTPS inclus

⚠️ Règles d'or de la sécurité front-end

• JAMAIS de secrets (clés API, tokens) dans le code client
• TOUJOURS sanitiser les données utilisateur avant insertion DOM
• PRÉFÉRER textContent à innerHTML par défaut
• ACTIVER une CSP stricte en production
• VALIDER côté serveur même si validation côté client
• HTTPS obligatoire pour tout en production
• NE PAS stocker de tokens sensibles dans localStorage

📚 Lexique des concepts clés

ES Modules

Système de modules natif JavaScript (import/export) sans bundler.

<script type="module" src="app.js">
import { func } from './module.js';

SPA (Single-Page App)

Application web qui charge une seule page HTML et met à jour dynamiquement le contenu sans rechargement.

Avantages : Navigation fluide, expérience app-like

Code-Splitting

Technique pour diviser le code en plusieurs fichiers chargés à la demande.

import('./page.js').then(m => m.render())

Résultat : Chargement initial plus rapide

Pub/Sub (Publisher/Subscriber)

Pattern de communication où les composants s'abonnent à des événements.

Permet un découplage fort entre modules

XSS (Cross-Site Scripting)

Injection de code malveillant dans une page web via des données utilisateur non filtrées.

⚠️ Peut voler cookies, tokens, données sensibles

CSP (Content Security Policy)

Header HTTP qui définit quelles ressources le navigateur peut charger.

Protection puissante contre XSS et injection de code

DOMPurify

Bibliothèque de sanitisation HTML qui nettoie le code malveillant.

const clean = DOMPurify.sanitize(dirty);

Service Worker

Script qui s'exécute en arrière-plan pour gérer le cache et fonctionnalités offline.

⚠️ Nécessite HTTPS. À utiliser avec précaution.

Lazy Loading

Chargement différé des ressources uniquement quand elles sont nécessaires.

<img loading="lazy" src="image.jpg">

SRI (Subresource Integrity)

Hash cryptographique qui vérifie l'intégrité d'un fichier externe.

<script src="..." integrity="sha256-..."
crossorigin="anonymous">

ARIA (Accessible Rich Internet Applications)

Attributs HTML qui améliorent l'accessibilité pour les lecteurs d'écran.

aria-label="Menu principal"
role="navigation"

BEM (Block Element Modifier)

Convention de nommage CSS pour structure et lisibilité.

.card { }
.card__title { }
.card--featured { }
